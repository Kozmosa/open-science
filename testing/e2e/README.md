# AINRF E2E Testing Setup Guide

## Quick Start (5 minutes)

### 1. Check Prerequisites

```bash
./testing/e2e/check-prereqs.sh
```

This verifies Docker, Node.js, and Playwright MCP availability.

### 2. Install Missing Dependencies

```bash
# Docker (if not installed)
curl -fsSL https://get.docker.com | sh
sudo usermod -aG docker $USER
# Log out and back in for group change

# Node.js 22+ (if not installed)
curl -fsSL https://deb.nodesource.com/setup_22.x | sudo -E bash -
sudo apt-get install -y nodejs

# Playwright MCP (first-time — npx handles this automatically)
npx --yes @playwright/mcp@latest --help
```

### 3. Build and Start Test Environment

```bash
cd testing/e2e
./run.sh up
```

This:
1. Builds the AINRF Docker image (first run: ~5 min, subsequent: ~30s cached)
2. Starts ainrf + nginx containers on ports 8198/8199
3. Seeds 3 test users (admin, alice, bob)
4. Prints credentials to terminal

### 4. Configure Your Agent's MCP Server

For **Claude Code**:
```bash
claude mcp add playwright -- npx @playwright/mcp@latest --headless --isolated
```

For **Codex CLI** (`~/.codex/config.toml`):
```toml
[mcp_servers.playwright]
command = "npx"
args = ["@playwright/mcp@latest", "--headless", "--isolated"]
```

For **Oh-My-Pi** — the MCP config is at `testing/e2e/config/mcp-servers.json`.
Copy it to wherever your agent reads MCP server configuration.

### 5. Run Agent Tests

Point your coding agent at:
- The **checklist**: `testing/e2e/CHECKLIST.md`
- The **agent prompt**: `testing/e2e/AGENT_PROMPT.md`
- The **credentials**: `testing/e2e/credentials.json` (generated by `run.sh up`)

Example agent invocation:
```
Read testing/e2e/AGENT_PROMPT.md and testing/e2e/CHECKLIST.md.
The AINRF E2E environment is running at http://localhost:8198 (frontend)
and http://localhost:8199 (backend). Execute all smoke tests and core
feature tests. Write results to testing/e2e/results/report-<timestamp>.md.
```

### 6. Tear Down

```bash
cd testing/e2e
./run.sh down
```

## Architecture

```
testing/e2e/
├── docker-compose.yml      # Ephemeral test containers (tmpfs, no persistence)
├── run.sh                  # Up/down/status/logs/rebuild commands
├── check-prereqs.sh        # Verify Docker, Node, Playwright MCP
├── CHECKLIST.md            # Test scenarios (smoke, features, perf, security)
├── AGENT_PROMPT.md         # System prompt for the test agent
├── README.md               # This file
├── config/
│   ├── nginx-e2e.conf      # Simplified nginx (no TLS, no auth proxy)
│   ├── seed.py             # Create test users + write credentials
│   └── mcp-servers.json    # Playwright MCP config for agents
└── results/                # Agent writes test reports here
```

## Port Reference

| Port | Service | URL |
|------|---------|-----|
| 8198 | nginx (frontend + API proxy) | http://localhost:8198 |
| 8199 | FastAPI backend (direct) | http://localhost:8199 |

Override with `AITestPort` and `AIWebPort` env vars.

## CI Integration

```yaml
# GitHub Actions example
- name: Start E2E environment
  run: testing/e2e/run.sh up

- name: Run agent E2E tests
  env:
    ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }}
  run: |
    # Point agent at the environment and run checklist
    claude --print "Read testing/e2e/AGENT_PROMPT.md..." 

- name: Tear down
  if: always()
  run: testing/e2e/run.sh down
```

## Troubleshooting

### "Port 8199 already in use"
Either a previous run didn't clean up, or another service uses the port:
```bash
./run.sh down          # clean up
AITestPort=8299 AIWebPort=8298 ./run.sh up  # use different ports
```

### "Backend did not become healthy"
```bash
./run.sh logs           # check container logs
docker exec ainrf-e2e curl -v http://localhost:8000/health  # check inside container
```

### "Cannot connect to Playwright MCP"
The MCP server needs Node.js and a first-time npx install:
```bash
npx --yes @playwright/mcp@latest --help  # force install
```

### Docker permission denied
```bash
sudo usermod -aG docker $USER
# Log out and back in
```
