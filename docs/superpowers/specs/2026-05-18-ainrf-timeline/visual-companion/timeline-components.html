数据获取策略

Query Endpoint 时机 polling
sessionsQuery GET /sessions?project_id=&status= 页面加载 + filter 变化 15s
sessionDetails[] GET /sessions/{id} (per session) sessions 加载后 batch 请求 30s
优化：使用 useQueries 批量加载 session details，渲染 Gantt 行时按需取 attempt 数据。首次加载 sessions 列表（轻量）→ 再批量拉取每个 session 的 attempts。

关键计算：Attempt 时间轴定位

// 对所有 attempts 计算全局时间范围
minTime = Math.min(...allAttempts.map(a => a.started_at))
maxTime = Math.max(...allAttempts.map(a => a.finished_at))
span = maxTime - minTime

// 每个 attempt 的位置和宽度
left = (a.started_at - minTime) / span * 100 // %
width = Math.max(1, (a.finished_at - a.started_at) / span * 100) // %

// 自适应时间轴标签单位
if (span <= 86400000): unit = "hour" // ≤ 24h
elif (span <= 604800000): unit = "day" // ≤ 7 days
else: unit = "week"
与 humanize2 renderGantt 中的计算逻辑一致。max(1, width) 确保极短的 attempt 至少可见 1%。

Gantt Row 行内交互

Hover → Tooltip 浮层显示 attempt 详情：
status · duration · cost · task link · attempt # · intervention_reason
Click attempt bar → 跳转到 /tasks/{task_id}（如果 attempt 关联了 task）
Click session label → 跳转到 /sessions?selected={session_id}
Scroll/zoom → 水平滚动查看更多时间范围。可以用 Ctrl+滚轮 缩放时间轴（调整 span）
