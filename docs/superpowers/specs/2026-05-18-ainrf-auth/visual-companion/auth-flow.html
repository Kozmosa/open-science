多用户鉴权 — Auth Flow & Middleware

Section 2/5 · JWT 签发/验证 + 注册审批 + 前端登录页面 + CLI login

1. JWT 设计

Access Token（短期，15min）：
Header: {alg: "HS256", typ: "JWT"} | Payload: {sub: user_id, role: "admin|member", exp, iat}

Refresh Token（长期，7d）：
随机 UUID → SHA256 hash 存入 refresh_tokens 表

JWT Secret: ~/.ainrf/jwt_secret（首次启动自动生成 64-byte random hex）

2. Auth API Endpoints

Method Path Auth 说明
POST /auth/register 无 注册（默认 pending）
POST /auth/login 无 登录 → access + refresh token
POST /auth/refresh Refresh Token 刷新 access token
POST /auth/logout Access Token 删除 refresh token
GET /auth/me Access Token 当前用户信息

3. Admin 管理 API（仅 Admin 可调用）

Method Path 说明
GET /admin/users 用户列表（含 pending）
PATCH /admin/users/{id} 激活/禁用/改 role
PUT /admin/users/{id}/password 重置密码

4. Middleware 改造

替换现有 build_api_key_middleware → build_jwt_auth_middleware：
# 无需鉴权的路径
_EXEMPT = {"/health", "/docs", "/openapi.json", "/redoc",
           "/auth/login", "/auth/register", "/auth/refresh"}

# 从 Authorization header 提取 Bearer token → 解码 JWT
# → 验证 exp + 查 user 是否 active
# → request.state.current_user = {id, username, role}
# → 失败返回 401
Route handler 获取当前用户：
def _get_current_user(request: Request) -> dict:
    return request.state.current_user

6. CLI login（`ainrf login`）

$ ainrf login
Username: alice
Password: ******
Logged in as alice (member). Token saved to ~/.ainrf/token
CLI 将 access + refresh token 写入 ~/.ainrf/token。后续 CLI 命令（ainrf serve、ainrf run 等）自动携带。
