2. 各 Route 改造要点

Route List 过滤 Create/Write 检查
/projects Admin: 全部。Member: owner OR collaborator 任意登录用户可创建；自动设 owner=self
/tasks Admin: 全部。Member: owner OR project_collaborator create: 需要 project access。cancel/pause/resume: 需要 owner 或 project member
/environments Admin: 全部。Member: environment_access 列表 create/update/delete: Admin only
/workspaces Admin: 全部。Member: owner OR collaborator project 任意登录用户可创建；自动设 owner=self
/sessions Admin: 全部。Member: owner_user_id=self owner or project member
/terminal 按 user_id 过滤（现有逻辑保留） 替换 X-AINRF-User-Id → JWT user_id
/skills Admin: 全部。Member: 全部（共享 skill registry） import: Admin only
/resources Admin: 全部。Member: 用户有访问权的 env —

4. 前端适配点

AuthContext（新）： 登录后持有 user 对象 {id, username, role}。refresh 逻辑自动处理。

Layout.tsx 改动：
  • 未登录 → 重定向到 /login
  • 已登录但 pending → 显示 "等待审批" 页面
  • 顶部显示当前用户名 + 角色 + Logout 按钮
Settings.tsx 改动：
  • Admin 看到 "Users" tab（用户管理）
  • Admin 看到 "Environments" tab 中的权限组 UI
  • 普通用户看不到这些 tab
ProjectCard / ProjectDetail： 增加 "Collaborators" 区域，owner 可添加/移除 member/viewer。

TasksPage / SessionsPage： 不需要 UI 改动，数据自动按权限过滤。
