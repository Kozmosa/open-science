## 2026-05-24 文献追踪端到端测试与堵点修复

### 测试主题：Personalized Federated Learning

**发现堵点：**
1. API Key 环境变量名不匹配（`fetcher.py` 读 `ANTHROPIC_API_KEY`，settings.json 存 `ANTHROPIC_AUTH_TOKEN`）
2. arXiv 查询不加引号，返回大量不相关论文（`personalized federated learning` 被解析为 OR 语义）
3. LLM 返回的 JSON 有 trailing comma，Python 3.13 `json.loads()` 严格解析失败
4. 创建订阅后不自动触发论文获取
5. 前端 Refresh 按钮仅刷新数据库查询，不触发 arXiv 获取
6. 前端缺少 `POST /subscriptions/{id}/fetch` 的 API 封装

**修复内容：**
- `src/ainrf/literature/fetcher.py`：添加 `ANTHROPIC_AUTH_TOKEN` 备选；关键词短语加双引号精确匹配；使用 `json_repair` 库健壮解析 LLM JSON
- `pyproject.toml` + `uv.lock`：添加 `json_repair` 依赖
- `frontend/src/api/endpoints.ts`：添加 `triggerLiteratureFetch` API 封装
- `frontend/src/components/literature/SubscriptionSidebar.tsx`：创建订阅成功后自动调用 `triggerLiteratureFetch`
- `frontend/src/components/literature/PaperFeed.tsx`：Refresh 按钮先触发后台获取再刷新列表

**验证：**
- 后端：137 个前端测试通过，后端 lint 通过，pytest 通过（除已有失败的 `test_cli.py`）
- arXiv 查询：`"personalized federated learning"` 返回 3 篇高度相关论文
- LLM 摘要：Kimi API 成功生成中文标题 + 3 条概要 + 实践提醒
