## 01:30 CST | fix: harden Agent SDK transport and ClaudeCodeEngine session handling

### Background

生产环境 task `f5ad245007e3` (Agent SDK 引擎, reproduction mode, PDF input) 报错:

```
[stderr] Failed to decode JSON: JSON message exceeded maximum buffer size of 1048576 bytes
```

用户给定的是 PDF 路径, 为什么会受 JSON buffer 大小限制? 为什么一次工具调用失败整个 session 就终止? retry 为什么也恢复不了?

### Root Cause Analysis

1. **JSON buffer 1MB 限制**: Agent SDK 的 `subprocess_cli.py` 传输层有 `_DEFAULT_MAX_BUFFER_SIZE = 1MB`。CLI 通过 `--output-format stream-json` 协议将 tool_result (包含 PDF 提取文本) 作为单行 JSON 返回给 SDK。大 PDF (50+ 页) 的提取文本+JSON 包装轻松超过 1MB → `SDKJSONDecodeError` → stdout 读取循环终止 → session 失败。不是 PDF 路径过 stdin 管道, 而是 Read 工具的**输出**通过 JSON 管道时超限。

2. **一次失败 → 全 session 崩溃**: `_read_messages_impl` 是单线程逐行 JSON 解析循环, buffer 超限直接 raise, 无跳过/恢复机制。

3. **Retry 也恢复不了 (AgentSdkEngine)**: resume 同一个 session → transcript 重放同一大 PDF → 同样溢出。Fresh-session 兜底丢失全部 tool 历史, 等于从头再来。

4. **ClaudeCodeEngine (`-p` 模式) 不受此影响**: 纯文本 stdout, 不做 JSON 解析, 无 buffer 限制。但它也不支持 retry/continue (send_input 直接抛异常, --no-session-persistence 阻止 session 写入)。

5. **容器 `~/.claude/CLAUDE.md` 对 SDK 子进程不可见**: SDK 的 `materialize_resume_session()` 和 `_run_query()` 都重定向 `CLAUDE_CONFIG_DIR` 到临时目录, 但只复制凭证文件 (`.credentials.json`, `.claude.json`), 不复制 `CLAUDE.md`。

### Changes

### ClaudeCodeEngine Retry Flow (After)

```
retry / send_input
  → send_input() enqueue last user message
  → start() called
    → has session_id?
      ├─ Yes → claude --resume <id> -p "pending message"
      │         ├─ Success → retain session_id, continue
      │         └─ Failure → clear session_id, fall through ↓
      └─ No  → claude -p --session-id <task_id> "[prior context] + message"
                → store session_id for next resume
```

### Performance Note

30MB `max_buffer_size` 无性能负担: `json_buffer` 是暂态峰值, 只在接收超大消息时分配, `json.loads()` 成功后立即 `= ""` 释放。正常消息从未超过几 KB。单次 6-10MB 的 `json.loads()` 耗时约 30-80ms, 远小于一次失败 session 白费的 token 成本。
