# 2026-05-19 Work Log

## 22:38 changelog: Fix project canvas edge connections

Fixed two bugs on the project canvas page:

1. **Manual edge connections disappeared on mouse release**: Root cause was that `Handle` components in `TaskNode.tsx` lacked explicit `id` props, causing ReactFlow v12 to not properly resolve handle endpoints. Also, the `useEffect` was blindly overwriting `flowEdges` with `initialEdges`, removing manually-added edges on re-render.

2. **Auto-connect edges by `created_at` order never appeared**: Root cause was stale `task_edges` rows in the database (9 old edges referencing a mix of archived/active tasks). Since `edges.length > 0`, the auto-connect logic in `initialEdges` was completely skipped.

Changes:
- `frontend/src/components/project/TaskNode.tsx`: Added explicit `id="target"` and `id="source"` to Handle components
- `frontend/src/components/project/ProjectCanvas.tsx`:
  - `initialEdges` memo: now filters stale backend edges (those whose source/target tasks aren't in the visible tasks set). If all backend edges are stale, falls through to auto-connect logic
  - Edge sync effect: replaced blind `setFlowEdges(initialEdges)` with merge logic that preserves manually-added edges tracked in `manualEdgeIds` ref
  - `onConnect`: uses ReactFlow's `addEdge` utility, tracks manual edge IDs for merge preservation
  - Added `connectionLineStyle` and `defaultEdgeOptions` props to `<ReactFlow>` for consistent styling

Verification: TypeScript `tsc -b` passed cleanly. All 119 frontend tests pass (2 skipped, unchanged).
