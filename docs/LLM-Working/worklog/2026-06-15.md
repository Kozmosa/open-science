## ~16:30 [observability] Phase 0 — critical fixes

- Migrated hand-rolled in-memory metrics (`list[float]` leak, custom text render) to `prometheus_client` SDK (Counter/Histogram/Gauge with declared label names). All 19 pre-declared metrics migrated; `reset_metrics()` selectively unregisters only own collectors to preserve Python runtime GC/platform metrics. Added `prometheus-client>=0.22.0` to pyproject.toml.
- Fixed Litefuse trace-generation disconnection: `start_trace()` now uses `as_type="trace"` (root); `record_generation()` and `record_span()` pass `trace_context={"trace_id": trace_id}` so LLM calls nest under their parent trace in the UI instead of appearing as orphaned observations.
- Added `exception_handler.py` middleware: catches unhandled exceptions → returns structured JSON `{error_code, message, request_id}`. Does NOT catch `HTTPException` (FastAPI handles) or `CancelledError` (must propagate). Registered as innermost middleware in `app.py`.
- Updated `request_context.py` to accept upstream `X-Request-ID` (reuse if present, generate UUID otherwise) and parse W3C `traceparent` for distributed tracing. All bound contextvar keys explicitly unbound in `finally` block.
- Fixed `QueryTimer` SQL text propagation: `instrument_connection()` stores `last_sql` on `conn._ainrf_last_sql`; `QueryTimer.sql` property auto-reads from connection via fallback chain (explicit sql → conn → "(unknown)").
- Validation: all 579 unit tests passed, all 31 metrics+observability tests passed.
